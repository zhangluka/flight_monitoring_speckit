# Research: 航班搜索数据源切换为真实接口

**Feature**: 002-real-api-integration  
**Date**: 2026-03-05

本 research 解决「如何将 mock 更换为真实可用的接口数据」的设计决策；具体第三方 API 选型与合规由 001-flight-search 的 research 覆盖，此处不重复。

---

## 1. 数据源配置策略

### Decision

- **驱动方式**：使用**环境变量**决定当前数据源（mock vs 真实 API）。例如 `FLIGHT_DATA_SOURCE=mock` 或 `FLIGHT_DATA_SOURCE=real`；未设置或无效时视为 mock，保证搜索仍可用。
- **真实 API 配置**：当 `FLIGHT_DATA_SOURCE=real` 时，至少需要真实 API 的端点与认证信息（如 `FLIGHT_API_BASE_URL`、`FLIGHT_API_KEY`），具体键名与供应商协议一致，在 quickstart 与实现中说明。
- **单实例**：仅支持一个真实 API 实例；不实现多数据源、主备或故障自动切换。

### Rationale

- 规格明确「配置缺失或无效时回退 mock」「通过配置切换、无需改代码」；环境变量与 001 quickstart 已约定一致，部署与本地开发均可复用。
- 单一真实 API 满足当前范围，避免过度设计；后续若有主备需求可单独迭代。

### Alternatives Considered

| 选项 | 说明 | 未选原因 |
|------|------|----------|
| 配置文件（JSON/YAML） | 将数据源与 API 配置放在仓库或挂载文件中 | 环境变量与 001 一致，且密钥更易与 CI/部署集成，不落库 |
| 多数据源 + 主备 | 配置多个 API，主失败时自动切备用 | 规格明确「单一即可」，不做本 feature |
| 仅真实 API、无 mock 回退 | 不配置则直接报错 | 规格要求配置缺失时回退 mock，便于开发/演示 |

---

## 2. BFF 数据源抽象

### Decision

- **单一调用入口**：BFF 路由（如 `app/api/flights/search/route.ts`）内根据配置选择「mock 实现」或「真实 API 客户端」；对外请求/响应契约与 001 的 `contracts/flight-search-api.md` 一致，前端无感知。
- **实现方式**：在 BFF 层抽象「获取航班列表」为同一接口（如 `getFlights(params): Promise<Flight[]>`），两实现分别为：从内存 mock 返回；或请求真实 API 后做响应映射。不要求独立 npm 包或插件体系。

### Rationale

- 规格要求「数据源由配置决定、不写死」；抽象为单一接口便于测试（可注入 mock/real）与后续替换供应商。
- 保持 BFF 对外契约不变，符合 001 契约与前端现有调用方式。

### Alternatives Considered

| 选项 | 说明 | 未选原因 |
|------|------|----------|
| 插件/策略注册表 | 运行时注册多个数据源实现，按名称选择 | 当前仅 mock + 单真实 API，无需插件复杂度 |
| 前端直接调真实 API | 由前端根据配置选择请求 BFF 或第三方 | 宪章与 001 要求 BFF 转发、密钥不暴露，不采纳 |

---

## 3. 真实 API 响应映射与失败处理

### Decision

- **映射策略**：真实接口返回的数据在 BFF 内**尽力映射**为现有展示契约（Flight：航班号、起降时间、出发地/目的地、舱位与价格）；缺失或无法映射的字段用占位或隐藏（如空串、0、或省略该字段），并**记录日志**便于排查，不向前端暴露原始错误或堆栈。
- **失败处理**：真实接口超时、4xx/5xx 或网络错误时，BFF 返回用户可理解的错误信息（契约中的 `error` + 可选 `code`）；**不实现** mock/缓存降级。同时**记录运维可用日志**（至少错误类型，可选请求标识如 origin/destination/date），不要求额外指标或追踪系统。
- **限流/配额**：若真实接口返回限流或配额错误，统一归入「接口错误」处理：用户端友好文案（如「请求过于频繁，请稍后再试」），服务端记录日志；凭证与配额详情不暴露给前端。

### Rationale

- 规格明确「尽力映射」「配置缺失回退 mock」「真实接口失败仅错误提示、不降级」「失败时记录运维日志」；上述决策与之一致。
- 映射与日志在 BFF 内完成，便于后续更换供应商或调整字段而不影响前端。

### Alternatives Considered

| 选项 | 说明 | 未选原因 |
|------|------|----------|
| 严格契约：不符合则整次失败 | 任一条或任一段不符合即返回 5xx | 规格要求尽力映射、部分展示，采纳更友好体验 |
| 真实 API 失败时自动降级 mock | 超时/5xx 时回退到内存 mock | 规格明确本 feature 不要求降级 |

---

## 4. 合规与记录

- 真实 API 的供应商、接口文档与使用条款以 001 的 research 及最终选型为准；本 feature 仅在 BFF 内实现「可配置调用 + 映射 + 日志」。
- 环境变量键名、示例与必填项在 `quickstart.md` 与 `contracts/data-source-config.md` 中说明；API Key 仅存服务端，不提交版本库。
