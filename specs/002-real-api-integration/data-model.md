# Data Model: 航班搜索数据源切换为真实接口

**Feature**: 002-real-api-integration  
**Date**: 2026-03-05

本 feature 在 001 的数据模型基础上，增加「数据源配置」相关概念；航班、舱位、搜索条件等与 001 保持一致，见 [001-flight-search/data-model.md](../001-flight-search/data-model.md)。

---

## 1. 数据源配置（运行时常量）

BFF 在启动或处理请求时读取的配置，决定当前使用 mock 还是真实 API；不持久化到数据库，来源为环境变量或部署配置。

| 字段/概念 | 说明 | 校验/规则 |
|-----------|------|-----------|
| 数据源模式（mode） | `mock` 或 `real` | 未设置或无效时视为 `mock`，保证搜索可用 |
| 真实 API 端点（baseUrl） | 真实航班搜索 API 的 base URL | 当 mode=real 时必填（或由供应商文档约定） |
| 认证（如 apiKey） | 真实 API 所需密钥或 token | 仅服务端持有，不暴露给前端；键名与供应商一致 |

- **身份与唯一性**：运行时仅存在一份「当前数据源配置」；单实例，无多数据源或主备。
- **与 BFF 行为**：mode=mock 时 BFF 使用内存 mock；mode=real 时 BFF 请求 baseUrl 并注入认证，再将响应映射为 001 的 Flight[]。

---

## 2. 沿用 001 的实体（无变更）

- **搜索条件（SearchParams）**：出发地、目的地、日期；校验规则同 001。
- **航班（Flight）**：航班号、起降时间、出发地/目的地、舱位列表；来自 BFF 归一化后的结构。
- **舱位（Cabin）**：舱位名称、价格；与航班一对多。

真实 API 的原始响应在 BFF 内映射为上述结构；缺失或无法映射的字段用占位或隐藏，并记录日志。

---

## 3. 与规格的对应

- **FR-001 / FR-002**：数据源配置对应「数据源模式 + 真实 API 端点/认证」；配置缺失或无效时回退 mock。
- **FR-003**：映射目标为 001 的 Flight/Cabin 结构；尽力映射，缺失字段占位或隐藏并记录日志。
- **FR-005**：认证信息仅存在于服务端配置（环境变量等），不暴露给前端。
